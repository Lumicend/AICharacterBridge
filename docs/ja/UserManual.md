## 1. キーボードショートカット一覧

| キー | 機能 | 有効な場面 |
|------|------|-----------|
| **K** | Character Card & World Editor の開閉 | メインゲーム中のみ |
| **L** | TalkScene Chat UI の開閉 | TalkScene（会話シーン）中のみ |

> **補足**：K キーと L キーはプラグイン設定から変更できます。

## 2. プラグイン設定（F1キー）

F1キーでプラグイン設定画面を開き、「**AI Character Bridge**」セクションを展開すると、以下の設定項目が表示されます。

### 2-1. General セクション

| 設定名 | デフォルト値 | 説明 |
|--------|-------------|------|
| **AI Client Type** | Ollama | 使用するAIクライアントを選択します。現在 `Ollama` と `LM Studio` が利用可能です |
| **Language** | English | AIが返答する言語を指定します（例：`Japanese`、`English`） |
| **Max Logs Per Heroine** | 20 | ヒロインごとに保存する会話ログの最大件数。`0` に設定するとログが保存されなくなります |

### 2-2. Keyboard Shortcuts セクション

| 設定名 | デフォルト値 | 説明 |
|--------|-------------|------|
| **Toggle Character Card Editor UI** | K | Character Card & World Editor の開閉キー |

TalkScene Chat UI のキーは後述の「TalkSceneChat」セクションで設定します。

### 2-3. Client - Ollama セクション

| 設定名 | デフォルト値 | 説明 |
|--------|-------------|------|
| **1. Model Name** | （空欄） | 使用する Ollama モデル名（例：`gemma3:12b`）。**必ず設定してください** |
| **2. Timeout (seconds)** | 300 | リクエストのタイムアウト秒数 |
| **3. Think Option** | Default | `think` フィールドのリクエスト付与を制御します（下記参照） |
| **4. LLM Options** | （空欄） | LLM生成オプションをJSON形式で記述します（下記参照） |

**Think Option について**

QwQ や DeepSeek-R1 など、拡張思考（Thinking）機能をサポートするモデルで使用します。

| 値 | 動作 |
|----|------|
| `Default` | `think` フィールドをリクエストに含めません |
| `True` | リクエストに `"think": true` を追加します |
| `False` | リクエストに `"think": false` を追加します（思考を無効にして応答を高速化します） |

**LLM Options の記述形式と例**

オブジェクト全体を囲む `{}` は**不要**です。Ollama API の `options` オブジェクト内のフィールドをそのまま記述します。

```
"temperature": 0.8,
"top_k": 40,
"top_p": 0.9
```

### 2-4. Client - LM Studio セクション

| 設定名 | デフォルト値 | 説明 |
|--------|-------------|------|
| **1. Base URL** | `http://localhost:1234` | LM Studio サーバーのベースURL |
| **2. Model Name** | （空欄） | 使用するモデル名。空欄の場合、LM Studio で現在ロード中のモデルが自動で使用されます |
| **3. Timeout (seconds)** | 300 | リクエストのタイムアウト秒数 |
| **4. LLM Options** | （空欄） | LLM生成オプションをJSON形式で記述します（下記参照） |

**LM Studio の LLM Options 記述例**

```
"temperature": 0.8,
"top_p": 0.9,
"max_output_tokens": 500
```

> **重要**：LM Studio ではトークン上限の指定に `max_tokens` ではなく **`max_output_tokens`** を使用してください。

### 2-5. TalkSceneChat セクション

| 設定名 | デフォルト値 | 説明 |
|--------|-------------|------|
| **Toggle UI Key** | L | TalkScene Chat UI の開閉キー |
| **Enable Favorability Update** | true | 会話内容をヒロインの好感度（favor）に反映するか |
| **Enable Arousal Update** | true | 会話内容をヒロインの興奮度（lewdness）に反映するか |

## 3. Character Card & World Editor（Kキー）

**K キー**（デフォルト）でウィンドウを開きます。**メインゲーム（フリーローム）中のみ使用できます。**

> **補足**：TalkScene（会話シーン）中に K キーを押すと、現在会話中のヒロインをリストの先頭に優先表示した状態でウィンドウが開きます。

### 3-1. ウィンドウの構成

ウィンドウは左右の2つのパネルに分かれています。

- **左パネル（セレクター）**：編集対象を選択します
  - `World`：World Setting の編集
  - `Player`：プレイヤーの Character Card の編集
  - ヒロイン名のリスト（`#番号 - ヒロイン名` の形式）：各ヒロインの Character Card の編集

- **右パネル（エディター）**：選択した対象のデータを編集します

ウィンドウのタイトルバーに `*` が表示されている場合は、未保存の変更があることを示しています。

### 3-2. 右パネル — エディタータブ

対象が **World** の場合は「**Description**」タブのみが表示されます。  
対象が **Player** または **Heroine** の場合は、以下の4つのタブが表示されます。

| タブ | 内容 |
|------|------|
| **Description** | キャラクターの説明・設定 |
| **Name** | AIに認識させるキャラクター名 |
| **Personality** | キャラクターの性格特性 |
| **Coordinate** | 衣装プリセットごとの服装・容姿説明文 |

#### Description タブ

キャラクターの基本的な設定・説明文を記述するフィールドです。キャラクターのバックグラウンド、身体的特徴、関係性などを自由に記述できます。  
World が選択されている場合は、ゲームの世界観・背景設定を記述します（詳細は「[5. World Setting について](#5-world-setting-について)」参照）。

#### Name タブ

AI に認識させるキャラクターの名前を設定します。このフィールドの名前がプロンプト内での呼び名として使用されます。

> **補足**：このフィールドが空欄の場合は、Koikatsu のゲーム内キャラクター名が自動的に使用されます。

#### Personality タブ

キャラクターの性格特性を記述します。どのような態度・価値観・話し方をするかを記述することで、AIの返答がより一貫したものになります。

#### Coordinate タブ

衣装プリセット（0〜6）ごとに、**Clothes**（服装説明）と **Appearance**（容姿説明）の2種類の説明文を設定できます。

| プリセット番号 | ゲーム内の衣装 |
|--------------|--------------|
| 0 | 学生服（校内） |
| 1 | 学生服（下校） |
| 2 | 体操着 |
| 3 | 水着 |
| 4 | 部活 |
| 5 | 私服 |
| 6 | お泊り |

**操作手順：**

1. 上段のボタン（`0: School` 〜 `3: Swimsuit`）または下段（`4: Club Activity` 〜 `6: Sleepwear`）でプリセット番号を選択します
2. `Clothes` / `Appearance` ボタンで編集するフィールドを選択します
3. テキストエリアに説明文を入力します
4. 他のプリセットやフィールドに切り替えると、現在の入力内容が自動的に一時保存されます
5. **Apply** ボタンを押すと、すべてのプリセット・フィールドのデータがセーブデータに反映されます

ここで設定した説明文は、Character Card の各テキストフィールドに `{{clothes}}` / `{{appearance}}` プレースホルダーとして記述しておくことで、プロンプト生成時に自動的に挿入されます（詳細は「[4-4. プレースホルダー機能](#4-4-プレースホルダー機能)」参照）。

### 3-3. ファイル入出力（Load JSON / Save JSON）

**Load JSON：**  
Character Card V2 / V3 形式の JSON ファイル、または World Setting の JSON ファイルをインポートできます。読み込み後は各タブの内容が更新されますが、**Apply を押すまでセーブデータには反映されません。**

**Save JSON：**  
現在編集中のデータを JSON ファイルとしてエクスポートします。ファイルはタイムスタンプ付きのファイル名で以下のディレクトリに保存されます。保存後、エクスプローラーが自動で開いて保存先が表示されます。

| 対象 | 保存先ディレクトリ |
|------|-----------------|
| Player / Heroine の Character Card | `(プラグインDLL と同じフォルダ)/CharacterCards/` |
| World Setting | `(プラグインDLL と同じフォルダ)/WorldSettings/` |

> **補足**：Character Card の Save JSON は Coordinate タブで設定した衣装説明文（CoordinateData）を含む完全なデータを出力します。

### 3-4. Apply ボタン・Clear ボタン

| ボタン | 動作 |
|--------|------|
| **Apply** | 右パネルの編集内容をセーブデータに反映します。**押さないと変更は失われます** |
| **Clear** | 右パネルの内容を空のデフォルト状態にリセットします（Apply するまでセーブデータには反映されません） |

> **重要**：
> - ウィンドウを閉じたり、別の対象を選択したりしても、Apply 前の変更はセーブデータに反映されません
> - Clear 後に元に戻す（Undo）機能はありません
> - Apply でセーブデータに反映しても、**ゲーム本体をセーブしないとゲーム終了時に消えます**

## 4. Character Card について

### 4-1. Character Card とは

Character Card は、AI にキャラクターの人格・背景・話し方を伝えるためのデータです。**Character Card V2 / V3** という汎用規格に準拠しており、他の AI チャットアプリ向けに作られたキャラクターカードをそのままインポートして使用できます。

プレイヤーキャラクター用とヒロイン用の両方に設定します。両方が設定されていないと TalkScene Chat の Talk ボタンが機能しません。

### 4-2. 対応フォーマット

| バージョン | 説明 |
|------------|------|
| **Character Card V2** | 標準的な形式。Character Card & World Editor で新規作成する場合はこちらが使用されます |
| **Character Card V3** | V2 の拡張版。ニックネームなどの追加フィールドをサポートします |

### 4-3. 各フィールドの役割

| フィールド | Character Card & World Editor のタブ | AIへの影響 |
|-----------|-------------------------------------|-----------|
| **name** | Name | プロンプト内でのキャラクター名として使用されます |
| **description** | Description | キャラクターの設定・背景説明としてプロンプトに含まれます |
| **personality** | Personality | 性格特性としてプロンプトに含まれます |
| **mes_example** | —（UI からは直接編集不可） | メッセージ例 |
| **first_mes** | —（UI からは直接編集不可） | 最初のメッセージ |
| **scenario** | —（UI からは直接編集不可） | シナリオ設定 |

> **補足**：`mes_example`、`first_mes`、`scenario` はデータとして欠落なく保持されますが、デフォルトのプロンプトテンプレートには含まれていません。Load JSON でインポートした場合もそのまま保存されます。

### 4-4. プレースホルダー機能

Character Card の各テキストフィールド（Description、Personality など）には、以下の2つのプレースホルダーを記述できます。

| プレースホルダー | 置換される内容 |
|-----------------|---------------|
| `{{clothes}}` | 現在の衣装プリセットに対応する Clothes 説明文（Coordinate タブで設定） |
| `{{appearance}}` | 現在の衣装プリセットに対応する Appearance 説明文（Coordinate タブで設定） |

これらはプロンプト生成時に、ゲームの現在の衣装プリセット番号に対応する説明文へ自動的に置換されます。未設定の場合は空文字列になります。

**記述例（Description フィールド内）：**
```
[容姿]
{{appearance}}

[服装]
{{clothes}}
```

Coordinate タブでプリセット 0（学校）の Appearance に「黒髪ポニーテール、茶色の瞳、すらりとした体型」、Clothes に「白いブラウスと紺色のスカートの制服」と設定しておけば、学校にいる場面でのプロンプトにその説明文が自動で挿入されます。

### 4-5. `{{user}}` / `{{char}}` プレースホルダーとの互換性

他のツール向けに作られた Character Card には、しばしば `{{user}}` や `{{char}}` という形式のプレースホルダーが含まれています。本プラグインはこれらを自動的に認識し、それぞれプレイヤー名・キャラクター名に置換します。

## 5. World Setting について

### 5-1. World Setting とは

World Setting は、ゲームの世界観・共通の背景設定を AI に伝えるためのデータです。すべてのヒロインとの会話に共通して適用されます。

Character Card & World Editor で「**World**」を選択し、Description タブに内容を記述します。内容が空欄の場合、World Setting はプロンプトから除外されます。

### 5-2. 記述例

```
舞台はコイカツの世界。主人公の男子生徒が通う学園を中心に展開する学園恋愛もの。
クラスメイトのヒロインたちと日常的な交流を深めていく話。
現代日本の一般的な学校文化・価値観が基本。派手なSF・ファンタジー要素はない。
```

詳細に書くほど AI がゲームの文脈をより正確に理解した返答を生成しやすくなります。設定したい世界観に合わせて自由に記述してください。

## 6. TalkScene Chat（Lキー）

**L キー**（デフォルト）でウィンドウを開きます。**TalkScene（会話シーン）中のみ使用できます。**

### 6-1. セッションの自動管理

TalkScene Chat のセッションは、TalkScene の開始・終了と完全に連動して自動管理されます。

| タイミング | 動作 |
|-----------|------|
| TalkScene 開始時 | 会話セッションが自動的に開始されます |
| TalkScene 終了時 | セッションが自動的に終了し、会話ログが保存されます。UI も自動的に閉じます |

L キーは TalkScene 中であれば何度でも開閉できます。UI を閉じても、セッションのデータ（会話ログなど）は保持されます。

**UI を開いたときの動作：**

UI を新たに開いた際、以下のデータが自動的に読み込まれます。

- **Prompt タブ**：保存済みのカスタムテンプレート（なければデフォルトテンプレート）
- **Context タブ**：現在のヒロイン用に保存済みの Context Note

### 6-2. Chat タブ

AI とのメッセージのやり取りを行うメインタブです。

**操作手順：**

1. テキストボックスにメッセージを入力します
2. **Talk** ボタンを押すと AI にリクエストが送信されます
3. AI の返答が届くと、ゲーム内のセリフとして画面上に表示されます

**各ボタンの説明：**

| ボタン | 説明 |
|--------|------|
| **Talk** | 入力したメッセージを AI に送信します。メッセージが空の場合は無効です |
| **Resend Last** | 直前に送信したメッセージをテキストボックスに復元します。再送信するには改めて Talk を押します |
| **特殊アクションボタン（緑）** | AI が特殊アクションを提案した場合に有効になります。クリックするとアクションを実行します（詳細は「[6-6. 特殊アクション](#6-6-特殊アクション)」参照） |

**AI 処理中について：**

AI の返答待ち中は「AI is thinking...」と表示され、Talk ボタンが無効になります。返答が届くまでの間は、ゲームの操作を控えることを推奨します。

**Talk ボタンを押したときに自動保存されるもの：**

- Prompt タブで編集中のプロンプトテンプレート
- Context タブで入力中の Context Note

これらは明示的に「Apply Changes」を押さなくても、Talk ボタンを押すことで保存されます。

### 6-3. Log タブ

過去の会話ログと現在のセッションログを確認できます。

**各ボタンの説明：**

| ボタン | 説明 |
|--------|------|
| **Delete Last Turn** | 現在のセッションの直近の1ターン（1回分のやり取り）を削除します |
| **Delete All Logs** | 現在のヒロインのすべての過去ログと現在のセッションログを削除します |

> **注意**：削除した内容を元に戻すことはできません。

**ログの表示形式：**

過去のログは「何日前」かでグループ化されて表示されます。現在進行中のセッションは「`--- Now ---`」として末尾に表示されます。

```
--- 2 days ago ---
[Afternoon Classes, conversation at school, Library, Wednesday]
Sosuke: "春野さん、一緒に図書室で勉強しない？"
Chika: "いいよ！じゃあ今日の放課後にしよう。"

--- Now ---
[Lunch Time, conversation at school, Cafeteria, Friday]
(会話が始まったところ)
```

### 6-4. Context タブ（Context Note）

現在の会話に関する補足情報（**Context Note**）を設定します。ここに書いた内容は AI に「現在の状況の補足説明」として伝わり、より文脈に合った返答を生成させることができます。

**使用例：**

```
今日は学園祭の前日。昨日、クラスの出し物についてちょっとした言い争いがあった。
プレイヤーは気まずい雰囲気を修復したいと思っている。
```

**各ボタンの説明：**

| ボタン | 説明 |
|--------|------|
| **Apply Changes** | 入力内容をセーブデータに保存します |
| **Reset** | セーブデータに保存された内容をテキストボックスに再読み込みします（未保存の入力を破棄します） |
| **Clear** | テキストボックスの内容を空にします（Apply Changes を押すまでセーブデータには反映されません） |

Context Note の内容は**ヒロインごとに独立して**保存されます。

> **補足**：Talk ボタンを押すと、その時点の Context Note が自動的に保存されます。毎回「Apply Changes」を押す必要はありません。ただし、UI を一度閉じて再度開くと、セーブデータから値が再読み込みされます。Apply を押さずに UI を閉じた場合、未保存の入力は失われます。

### 6-5. Prompt タブ

AI に送信するプロンプトのテンプレートを確認・編集できます。

**各ボタンの説明：**

| ボタン | 説明 |
|--------|------|
| **Apply Changes** | 編集したテンプレートをセーブデータに保存します |
| **Reset** | テンプレートをデフォルト（プラグイン内蔵）に戻します |

Talk ボタンを押すたびに、その時点のテンプレートが自動的に保存されます。

プロンプトテンプレートのカスタマイズについては「[8. プロンプトテンプレート](#8-プロンプトテンプレート)」を参照してください。

### 6-6. 特殊アクション

AI の返答には、会話の流れに応じた「**後続アクション（Post-Conversation Action）**」の提案が含まれます。このアクションが会話継続（`continue_conversation`）以外の場合、Chat タブのアクションボタンが**緑色**になり、クリックで実行できます。

**利用可能な特殊アクション一覧：**

| 表示テキスト | 説明 | 実行条件 |
|-------------|------|---------|
| Have Lunch Together | 一緒に昼食をとる | 昼食時間中、かつ今日まだ一緒に昼食をとっていない場合 |
| Join Club Activity Together | 一緒に部活をする | 部活時間中、かつヒロインが部員の場合 |
| Go Home Together | 一緒に帰宅する | 放課後の場合 |
| Study Together | 一緒に勉強する | — |
| Exercise Together | 一緒に運動する | — |
| Accompany Player | プレイヤーに同行する | — |
| Make Date Reservation | デートの約束をする | ヒロインがまだデート予約していない場合 |
| Consent to H Scene | H シーンへ移行する | — |

ゲームの現在の状況によって、条件を満たさないアクションは AI の選択肢から除外されます。

アクションを実行すると、その内容も会話ログに記録されます。

> **補足**：アクションボタンが緑色になっても、クリックするタイミングは自由です。提案を無視して会話を続けることも可能です。

### 6-7. 好感度・親密度・興奮度への影響

**好感度（favor）への影響** — `Enable Favorability Update` が `true` の場合のみ有効

AI の返答に含まれるユーザーへの印象評価（`impression_on_user`）に基づいて、ヒロインの好感度が変化します。

| 印象評価 | favor への変化 | 補足 |
|---------|--------------|------|
| `very_bad` | -4 | 最低値 0 |
| `bad` | -2 | 最低値 0 |
| `neutral` | ±0 | |
| `good` | +4 | favor が 100 かつ彼女（`isGirlfriend`）の場合は代わりに intimacy +1（最大 100） |
| `very_good` | +6 | favor が 100 かつ彼女（`isGirlfriend`）の場合は代わりに intimacy +1（最大 100） |

**興奮度（lewdness）への影響** — `Enable Arousal Update` が `true` の場合のみ有効

AI の評価（`is_aroused_by_conversation`）が `yes` の場合、ヒロインの興奮度（lewdness）が **+4** されます（最大 100）。

## 7. 会話ログ

### 7-1. ログの仕組み

TalkScene Chat で行われた会話は、TalkScene 終了時に自動的にログとして保存されます。ログはヒロインごとに管理され、ゲームのセーブデータに保存されます（ゲーム本体をセーブすることで永続化されます）。

保存されるログには以下の情報が含まれます。

- 会話が行われた曜日・時間帯・場所
- ユーザーとキャラクターの発言内容（セリフ・描写）
- 実行された特殊アクション（ある場合）

### 7-2. ログの上限設定

`Max Logs Per Heroine`（デフォルト：20）で保存件数の上限を設定できます。上限を超えると最も古いログから順に自動削除されます。

`0` に設定するとログが保存されなくなります。

### 7-3. ログがプロンプトに与える影響

保存されたログは次回以降の TalkScene でプロンプト内の会話履歴（`{{chat_log}}`）として自動的に挿入されます。これにより、AI は過去の会話を記憶した上で返答を生成できます。

ログが増えるとプロンプトが長くなり、LLM への入力トークン数が増加します。コンテキストウィンドウの小さいモデルを使用している場合や、動作が重くなってきた場合は、Max Logs Per Heroine の値を減らすか、Log タブで不要なログを削除してください。

## 8. プロンプトテンプレート

### 8-1. プロンプトテンプレートとは

AI に送信されるプロンプト（指示文）の雛形です。各種データは `{{プレースホルダー名}}` の形式でテンプレート内に記述されており、送信時に実際のデータに置換されます。

TalkScene Chat の **Prompt タブ**で内容を確認・編集できます。

### 8-2. プレースホルダー一覧

| プレースホルダー | 置換される内容 | 空の場合の挙動 |
|-----------------|---------------|--------------|
| `{{language}}` | 返答言語（General > Language の設定値） | — |
| `{{world_setting}}` | World Setting の説明文 | その行ごと削除されます |
| `{{user_name}}` | プレイヤーの Character Card の Name | — |
| `{{user_description}}` | プレイヤーの Character Card の Description | その行ごと削除されます |
| `{{user_personality}}` | プレイヤーの Character Card の Personality | その行ごと削除されます |
| `{{char_name}}` | ヒロインの Character Card の Name | — |
| `{{char_description}}` | ヒロインの Character Card の Description | その行ごと削除されます |
| `{{char_personality}}` | ヒロインの Character Card の Personality | その行ごと削除されます |
| `{{time_period}}` | 現在のゲーム内時間帯 | — |
| `{{week}}` | 現在のゲーム内曜日 | — |
| `{{location}}` | 現在のゲーム内場所 | — |
| `{{school_name}}` | 学校名（ゲームのセーブデータから取得） | — |
| `{{context_note}}` | Context Note の内容 | その行ごと削除されます |
| `{{chat_log}}` | 会話履歴（過去ログ＋現在セッションログ） | — |
| `{{user_message}}` | ユーザーが入力したメッセージ | — |
| `{{available_expressions}}` | 利用可能な表情名のリスト（JSON配列形式） | — |
| `{{available_chara_motions}}` | 利用可能なポーズ名のリスト（JSON配列形式） | — |
| `{{available_impressions}}` | 利用可能な印象評価値のリスト（JSON配列形式） | — |
| `{{available_post_actions}}` | 利用可能な後続アクションのリスト（JSON配列形式） | — |
| `{{user}}` | `{{user_name}}` の互換エイリアス | — |
| `{{char}}` | `{{char_name}}` の互換エイリアス | — |

### 8-3. テンプレートのカスタマイズ

Prompt タブでテンプレートを自由に編集できます。上記のプレースホルダーを使ってAIへの指示をカスタマイズできます。

**Reset ボタン**でデフォルトのテンプレートに戻ります。

カスタムテンプレートはゲームのセーブデータに保存されます。「Apply Changes」を押すか、Talk ボタンを押すことで保存されます。

> **注意**：テンプレートのカスタマイズは上級者向けの機能です。以下のプレースホルダーは AI が正しい形式の JSON を返すために重要です。削除・変更には注意してください。
> - `{{user_message}}`
> - `{{available_expressions}}`
> - `{{available_chara_motions}}`
> - `{{available_impressions}}`
> - `{{available_post_actions}}`

### 8-4. Context Note の活用

`{{context_note}}` プレースホルダーは、現在の会話の補足情報を柔軟に追加するための仕組みです。デフォルトテンプレートでは `<current_context>` セクション内に配置されており、ゲームの時間帯・場所情報と並んで AI に伝わります。

Context Note が空欄の場合、`{{context_note}}` の行はプロンプトから自動的に除去されます（余分な空行が残りません）。

## 付録B：トラブルシューティング（詳細版）

| 症状 | 確認・対処方法 |
|------|-------------|
| **K キーを押してもウィンドウが開かない** | メインゲーム中であるか確認してください。タイトル画面やキャラクターメイク画面では使用できません |
| **L キーを押してもウィンドウが開かない** | TalkScene（会話シーン）中であるか確認してください。行動メニューが表示された状態で試してください |
| **Talk ボタンを押しても何も起きない** | ① Ollama / LM Studio が起動しているか確認してください。② プレイヤーとヒロインの両方に Character Card が設定・Apply 済みか確認してください |
| **返答が返ってこない・タイムアウトする** | ① モデル名が正しく設定されているか確認してください。② Timeout 設定を増やしてみてください（大型モデルは生成に時間がかかります）。③ コマンドプロンプトで `ollama run <モデル名>` を試し、モデルが動作するか確認してください |
| **返答が英語で返ってくる** | F1 設定の `General > Language` を変更してください |
| **返答の形式がおかしい・エラーになる** | モデルの指示追従能力が不足している可能性があります。より高性能なモデルを試してください。7B 以上のパラメータ数のモデルを推奨します |
| **ヒロインの好感度が変化しない** | F1 設定の `TalkSceneChat > Enable Favorability Update` が `true` になっているか確認してください |
| **Character Card の設定が消えた** | Apply ボタンを押した後、ゲーム本体をセーブしているか確認してください。ゲームのセーブなしにゲームを終了すると設定が失われます |
| **Context Note や Prompt の変更が保存されない** | UI を閉じる前に「Apply Changes」を押すか、Talk ボタンを押してください |
| **特殊アクションボタンが表示されない** | AI が `continue_conversation` を選択している場合は表示されません。会話の流れによっては特殊アクションが提案されないことがあります |
