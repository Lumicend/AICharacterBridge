## 1. 必要なもの

以下のすべてを用意してください。

| 名前 | 備考 |
|------|------|
| **AICharacterBridge** | 本プラグイン |
| **BepInEx** | MODローダー |
| **KKAPI** | Koikatsu用API |
| **ExtensibleSaveFormat** | 拡張セーブデータ |
| **BepInEx.ConfigurationManager** | プラグイン設定のGUI管理 |
| **Newtonsoft Json.NET for Unity3D** | 本プラグインに同梱 ([GitHub](https://github.com/SaladLab/Json.Net.Unity3D)) |
| **ローカルLLMツール** | 本プラグインは [Ollama](https://ollama.com/)、およびオプションとして [LM Studio](https://lmstudio.ai/) に対応しています |
| **ローカルLLMモデル** | 上記ツールで動作する任意のモデル |

> **補足：ローカルLLMについて**  
> ローカルLLMツールの導入・使用方法は各ツールの公式ドキュメントを参照してください。モデルを取得後、ツールの標準機能で動作確認（LLMとの会話）を済ませてからゲームに進むことを推奨します。  
> 以降の手順は **Ollama** を使用する場合を前提に説明します。

## 2. インストール

### 2-1. 依存プラグインのインストール

以下のプラグインを、各リポジトリの導入手順に従ってインストールしてください。

- [BepInEx](https://github.com/BepInEx/BepInEx)
- [KKAPI](https://github.com/IllusionMods/IllusionModdingAPI)
- [ExtensibleSaveFormat](https://github.com/IllusionMods/BepisPlugins)（BepisPluginsに含まれます）
- [BepInEx.ConfigurationManager](https://github.com/BepInEx/BepInEx.ConfigurationManager)

### 2-2. AICharacterBridgeのインストール

1. [リリースページ（URL）](#) から最新版のZIPファイルをダウンロードします。
2. ZIPを解凍し、中身を Koikatsu のゲームフォルダにそのままコピーします。

## 3. 動作テスト

### 3-1. 事前準備（ゲーム起動前）

#### テスト用キャラクターカードのダウンロード

本プラグインはキャラクターの性格データを [Character Card V2](https://github.com/malfoyslastname/character-card-spec-v2) / [Character Card V3](https://github.com/kwaroran/character-card-spec-v3) 形式の JSON で管理します。

動作テスト用として、以下のサンプルデータを用意しています。ダウンロードしておいてください。

- [Sosuke_Kashiwagi.json（URL）](#)：プレイヤー用サンプルデータ
- [Chika_Haruno.json（URL）](#)：ヒロイン用サンプルデータ

> **補足：キャラクターカードとは**  
> キャラクターの名前・性格・口調・背景などを記述したJSONファイルです。ゲーム内キャラクターとは独立して管理されるため、ゲーム内での名前や性格は影響しません。このテストでは、登録したヒロインに「Chika Haruno」としての性格をAIに割り当てます。

#### 使用するモデル名のメモ

ゲームを起動する前に、Ollama で利用するモデルの名前を確認しておきます。

1. **Windowsキー + R** を押し、`cmd` と入力してコマンドプロンプトを開きます。
2. 以下のコマンドを実行します。
   ```
   ollama list
   ```
3. 取得済みのモデル一覧が表示されます。使用したいモデルの **Name** 列の値をコピーしておいてください。

> **重要**：ゲームのプレイ中も、Ollama は起動したままにしておいてください。Talk ボタンを押す際、Ollama がバックグラウンドで動作していることが必要です。

### 3-2. ゲーム内設定

#### ステップ1：プラグインのオプション設定

1. ゲームを起動します。
2. タイトル画面または任意のシーンで **F1キー**（BepInEx.ConfigurationManager のデフォルト）を押し、プラグイン設定画面を開きます。
3. 「**AI Character Bridge**」の項目が一覧にあることを確認します。
4. 「**Client - Ollama**」セクションを展開し、**「1. Model Name」** の欄に、先ほどメモしたモデル名を入力します。
5. 設定画面を閉じます。

#### ステップ2：テスト用のゲームデータを作成

1. 新規ゲームを開始します。
   - 学校名や男性キャラクターの設定はデフォルトのままで構いません。
   - **ヒロインを1人以上登録してください**（後のステップで話しかける相手になります）。
2. オープニングイベントが終了し、行動できる状態になったら、すぐに帰宅してください。
   - ※後ですぐやり直せるよう、この時点でゲームをセーブしておくことを推奨します。

#### ステップ3：キャラクターの性格データを設定

自宅に戻ったら、**「Kキー」(デフォルト設定)** で Character Card & World Editor ウィンドウを開きます。

ウィンドウの構成は以下のとおりです。

- **左パネル**：編集対象の選択（World / Player / ヒロイン名簿）
- **右パネル**：選択した対象の各種データ編集（Description / Name / Personality / Message Example / Coordinate）

---

**プレイヤーの性格を設定する**

1. 左パネルの「**Player**」をクリックします。
2. 右パネル上部の「**Load JSON**」ボタンを押し、ダウンロードした `Sosuke_Kashiwagi.json` を選択して読み込みます。
3. データが各タブに反映されていることを確認したら、「**Apply**」ボタンを押して確定します。

> **注意**：「Apply」を押す前に別の対象を選択したりウィンドウを閉じると、**変更が破棄されます**。必ずApplyを押してから次に進んでください。

---

**ヒロインの性格を設定する**

1. 左パネルのヒロイン名簿から、ステップ2で登録したヒロインをクリックします。
2. 「**Load JSON**」ボタンを押し、`Chika_Haruno.json` を選択して読み込みます。
3. 「**Apply**」ボタンを押して確定します。

---

以上で設定は完了です。Character Card & World Editor ウィンドウを閉じてください。

> **ヒント**：設定内容を次回のゲーム起動後も保持したい場合は、ここでゲームをセーブしておきましょう。この後すぐ動作テストを行うだけであれば、セーブは不要です。

#### ステップ4：次の日へ進む

次の日へ進んで学校へ向かいましょう。

---

### 3-3. 会話テスト

#### ステップ1：ヒロインに話しかける

校内を自由に行動できる状態になったら、設定したヒロインのいる場所へ向かい、話しかけます。

#### ステップ2：会話イベントを終わらせる

話しかけると通常の会話イベントが始まります。選択肢を選んでイベントを進め、**行動メニューが表示された状態**（アクション待ちの状態）にします。

#### ステップ3：TalkScene UIを開く

**デフォルト設定では「Lキー」** で TalkScene UI ウィンドウを開きます。

> **注意**：TalkScene中でなければ、Lキーを押してもウィンドウは開きません。  
> また、プレイヤーまたはヒロインの CharacterCard が未設定の場合、ウィンドウは開きますが、**Talk ボタンを押しても何も起きません**。その場合はステップ3の Apply が完了しているか確認してください。

ウィンドウには以下の4つのタブがあります。

| タブ | 内容 |
|------|------|
| **Chat** | メッセージの入力と送信 |
| **Log** | 会話履歴の確認・削除 |
| **Context** | ヒロインごとの補足情報（Context Note）の設定 |
| **Prompt** | AIへのプロンプトテンプレートの確認・編集 |

#### ステップ4：メッセージを送信する

1. 「**Chat**」タブが選択されていることを確認します。
2. テキストボックスに話しかける言葉を入力します。  
   例：`Hello, Chika!`
3. 「**Talk**」ボタンを押してOllamaにデータを送信します。

> **注意点**
> - 返答待ち中は「Talk」ボタンが無効になります。
> - 返答が届くまでの間は**ゲームを操作しないことを推奨します**。操作しても動かせますが、動作が不安定になる可能性があります。
> - 返答が返るまでの時間は、PCのスペック・使用モデル・Ollamaのモデル読み込み状況によって大きく変わります。初回はモデルの読み込みに時間がかかる場合があります。

#### ステップ5：返答の確認

正常に動作していれば、Chikaからの返答セリフがゲーム画面上に表示されます。また、会話の内容によって、ヒロインの好感度パラメータがわずかに変化します。

以上で、AI Character Bridge の基本的な動作テストは完了です。🎉

## トラブルシューティング

| 症状 | 確認すること |
|------|-------------|
| Lキーを押してもウィンドウが開かない | TalkScene中であるか確認してください |
| Talkボタンを押しても何も起きない | ① Ollamaが起動しているか確認してください（`ollama serve` が実行中であること）。② プレイヤーとヒロインの両方にCharacterCardが設定・Apply済みか確認してください |
| 返答が返ってこない・タイムアウトする | モデル名が正しく設定されているか確認してください。コマンドプロンプトで `ollama run <モデル名>` を試し、モデルが動作するか確認してください |
| 返答がおかしい / 英語で返ってくる | プラグイン設定の「**General > Language**」が正しく設定されているか確認してください（例：`Japanese`） |
